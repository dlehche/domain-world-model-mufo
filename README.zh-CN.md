# 领域世界模型：MUFO  
## 人体功能统一本体（MoveTips Unified Functional Ontology）

MUFO 是一个面向**人体功能**的领域世界模型，用于把功能结构、状态分布、风险边界与行动条件下的状态演化，表达为可计算、可审计、可治理的知识与规则系统。

## 为什么要做这个仓库

现有医疗与运动系统擅长疾病命名、流程执行和局部任务管理，但对于“人体功能”这一对象，长期缺少统一、稳定、可计算的表达框架。

疼痛、代偿、耐受下降、控制不稳、表现波动，往往在明确诊断之前就已经出现；但在康复、训练与健康管理实践中，这些问题常常被不同角色用不同语言理解和处理，导致记录不统一、推理不可复现、风险边界不稳定。

MUFO 试图解决的正是这个问题。

## MUFO 的定位

MUFO 不是诊断系统。  
MUFO 不是治疗指南。  
MUFO 也不是替代专业判断的黑盒模型。

MUFO 的定位是：

> 面向“人体功能世界”的统一知识与规则底座。

它的目标是支持：

- 人体功能的结构化表达
- 可解释的状态推理
- 明确的安全边界
- 可追溯的推理工件
- 可回写的纵向记录
- 受治理约束的人机协同

## 人体功能的工作定义

在 MUFO 中，人体功能被定义为：

> 人体在明确安全约束下，以有界的生理、力学与认知成本，稳定且高效完成任务的能力。

这个定义是**任务导向**的，而不是疾病优先的；也是**边界化**的，而不是理想化的。

## 核心判断主轴

MUFO 以 **任务–能力匹配模型（Task–Capacity Matching, TCM）** 作为核心判断主轴：

- 当任务需求大于身体能力时，疼痛、代偿、风险和表现不稳定更容易出现；
- 当身体能力覆盖任务需求时，功能更安全、更稳定，也更具提升空间。

这使得机械、神经、感觉、自主神经、呼吸等不同系统，能够放到同一套“需求–供给关系”里解释，而不是继续停留在碎片知识状态。

## 三层主结构

### 第一层：人体功能协作关系层
冻结人体功能之间的约束拓扑。

这一层定义哪些功能支撑哪些功能、哪些缺口会触发代偿、哪些关系会打开风险窗口。它定义结构，不直接计算状态。

### 第二层：人体状态解析引擎
把观测映射为功能状态分布。

这一层负责把评估输入、动作表现与结构化证据映射为状态、裕度、瓶颈与风险窗口，并生成证据链与护栏内候选行动。

### 第三层：人体功能世界模型
在结构与状态之上引入时间、行动与记忆。

这一层关注干预、负荷与策略变化如何推动状态迁移，并通过回写形成个体化的长期校准。

## 在世界模型栈中的战略位置

MUFO 不是分子模拟系统，不是纯视觉世界模型，也不是通用语言模型。

MUFO 的战略靶心是 **L4：人体功能动力学层**。  
这一层承接下层生物机制的约束，向上支撑任务表现与决策支持，是人体功能真正进入可计算、可审计、可治理状态的关键中轴。

## 与 MFLS 的关系

MUFO 是知识与推理底座。  
MFLS（木梯人体功能语言系统）是建立在 MUFO 之上的表达与执行语言，用于评估、干预、安全边界、记录、审计与发布治理。

两者共同构成一个以人体功能为中心的、可追溯、可约束、可演进的工程体系。

## 边界声明

MUFO 是一个知识表示与决策支持框架。

它不构成：
- 自动诊断系统
- 自动治疗系统
- 无约束推荐系统

所有输出都应在明确的安全护栏、证据链和人工审核机制下使用。
