# 领域世界模型：人体功能统一本体

## 执行摘要

“世界模型”的核心，不是“知道很多”，而是：用稳定的抽象表示描述当前状态，并在给定行动条件下预测后果，从而支持分层规划与决策。LeCun 在 2022 年的立场论文中明确提出，真正的自主机器智能需要可配置的预测性世界模型、多尺度抽象表示与分层规划，而不能仅仅依赖下一 token 预测。  
2026 年 3 月，Advanced Machine Intelligence（AMI）获得 10.3 亿美元融资，并公开把“世界模型、推理、规划、可控与安全”作为核心方向之一。这件事的意义不在于融资本身，而在于它再次把“世界模型路线”推回到 AI 的主舞台。

本文提出：MUFO（MoveTips Unified Functional Ontology）可以被理解为一个面向**人体功能**的领域世界模型。它的工程路径不是一开始就追求全尺度统一仿真，而是按三层递进展开：

1. 人体功能协作关系层：冻结人体功能的约束拓扑  
2. 人体状态解析引擎：把观测映射为状态分布与风险窗口  
3. 人体功能世界模型：在结构与状态之上引入时间、行动与记忆回写

MUFO 不是诊断系统，不是治疗指南，也不是替代专业判断的黑盒模型。它的目标，是把人体功能表达为一个**可计算、可审计、可治理、可演化**的知识与规则系统。

---

## 一、问题起点：人体功能知识为什么长期碎片化

现代医疗系统擅长疾病命名、病理解释和流程管理，却并不擅长把“人体功能”作为一个统一的推理对象来处理。

真实实践中，疼痛、代偿、耐受下降、控制不稳、表现波动，往往早于明确诊断出现。无论是在康复、训练还是健康管理场景里，这些问题通常都被拆散在不同话语体系中：

- 解剖讲结构  
- 生物力学讲受力  
- 康复讲功能缺陷  
- 训练讲动作表现  
- 量表讲任务结果  
- 病历讲症状记录  

知识并不少，真正缺少的是：**一套稳定、统一、可计算的人体功能坐标系。**

WHO 的 ICF 为功能与残障提供了国际分类框架，并于 2001 年通过 WHA54.21 获得正式国际认可。它对功能描述和跨系统沟通具有基础意义。  
但 ICF 的主要长处在于分类与语言统一，而不是在于为“观测 → 状态 → 约束 → 行动 → 回写”建立可执行的工程闭环。因此，在大量真实场景里，人体功能判断仍然停留在分散量表、经验语言和不可复用记录层面。

只要人体功能持续被拆散在不同语义系统中，相关判断就很难标准化、回放、审计和持续校准。

---

## 二、MUFO 的定位：人体功能世界的知识与规则底座

MUFO 的定位，不是“另一个评估表”，也不是“智能推荐壳子”，而是：

> 面向人体功能世界的统一知识与规则底座。

它的目标不是直接替代诊断、处方或人工判断，而是把人体功能表述成一个可表达、可推理、可约束、可审计的对象。

在 MUFO 中，人体功能被定义为：

> 人体在明确安全约束下，以有界的生理、力学与认知成本，稳定且高效完成任务的能力。 :contentReference[oaicite:3]{index=3}

这个定义有三个关键特征：

第一，它是**任务导向**的，而不是疾病优先的。  
第二，它是**边界化**的，而不是理想化的。  
第三，它天然要求把“能力、任务、约束、风险、结果”放进同一个判断框架里。

MUFO 的核心作用，是给人体功能建立一个统一的表示与推理底座，使其能够支持：

- 结构化表达  
- 状态解析  
- 风险边界控制  
- 推理可追溯  
- 记录可回写  
- 长期可治理  

从世界模型角度看，MUFO 的意义在于：它不是追求模拟全部生物学细节，而是要在人体功能这一层建立一个**足够稳定、足够实用、足够可审计**的中层表示系统。

## 三、第一层：人体功能协作关系层

第一层不是“功能清单”，而是**功能之间的约束结构**。

这一层回答的问题是：

- 哪个功能支撑哪个功能  
- 哪个功能不足会触发哪条代偿路径  
- 哪些关系形成稳定闭环  
- 哪些组合打开风险窗口  

从工程角度看，这一层本质上是一个**有向约束网络**。  
节点不是随意命名的标签，而是经过边界定义的功能单元。  
边也不是笼统的“有关联”，而是带有方向、类型、条件和治理属性的约束关系。

这一层的任务，是把人体功能世界的结构拓扑固定下来，而不是直接推断某个人当前处于什么状态。

如果这一层只做词表堆积和语义相邻，它最终会退化成普通知识图谱。  
只有当这一层被建设成带关系类型、条件表达、版本治理和约束边界的结构网络时，它才有资格成为世界模型的骨架。

这也是为什么本体治理原则重要。OBO Foundry 对关系复用、概念边界和规范化演进的强调，对 MUFO 有现实参考意义：不是为了追求“标准感”，而是为了防止系统在扩展中失去一致性。

---

## 四、第二层：人体状态解析引擎

真正的工程分水岭在第二层。

这一层的任务，不是继续扩充知识，而是把观测数据映射到功能网络中的**状态分布**上。也就是，把“看到什么”变成“现在处于哪里”。

它回答的问题是：

- 当前这个人处于什么功能状态  
- 哪些功能单元稳定，哪些处于边界，哪些已经失衡  
- 哪些代偿路径正在被启动  
- 哪些风险窗口已经被打开  
- 当前最该优先处理的是任务、策略还是能力缺口  

这一层不等于物理仿真。  
它是一个**约束驱动的状态推理系统**。

MUFO 在这里采用的核心判断主轴是 **Task–Capacity Matching（任务–能力匹配，TCM）**。  
它的基本思想可以简化为：

- 任务定义需求  
- 人体定义能力  
- 判断的核心，是能力是否在约束条件下覆盖需求  

最小表达可以写成：

**M = C − D**

其中：

- **C** = capacity，能力供给  
- **D** = demand，任务需求  
- **M** = margin，功能裕度  

这意味着，人体功能判断可以被表达为一种“需求–供给–边界”的关系判断，而不必一开始就声称解释全部机制。  
在工程上，这比直接求解全人体动力学要现实得多。

这一层最终输出的，不应只是分数或标签，而应包括：

- 当前状态分布  
- 风险窗口  
- 约束判断结果  
- 推理证据链  
- 下一步候选行动入口  

这也是 MUFO 与传统经验判断最大的不同：  
它试图把“感觉像是这样”的说法，转成“在什么观测、什么规则、什么版本下，被解释为这样”的可回放结构。

## 五、第三层：人体功能世界模型

只有在结构和状态都稳定以后，第三层才有意义。

世界模型在这里不是“知道更多内容”，而是获得了一种新的能力：

> 在结构与状态之上，引入时间、行动和记忆回写，从而模拟未来路径。

这一层回答的问题是：

- 如果采取某种干预，未来功能轨迹会怎样变化  
- 如果继续增加负荷，风险窗口是否会扩大  
- 如果改变策略，状态是否会回到稳定域  
- 个体差异如何改变演化路径  
- 哪些路径值得尝试，哪些路径应该回退  

从形式上说，它关注的是**行动条件下的状态迁移**。  
即：

**s(t+1) = f(s(t), a(t), u(t))**

其中：

- **s(t)** 是当前功能状态  
- **a(t)** 是行动、干预、负荷调整或策略变化  
- **u(t)** 是疲劳、疼痛、恢复、环境等上下文因素  

LeCun 2022 对世界模型的定义，核心不在“生成世界”，而在“用世界模型支撑规划”。JEPA 路线进一步强化了一点：真正重要的不是复现所有细节，而是在**表示空间里预测具有决策意义的变化**。

MUFO 把这个逻辑落实在人体功能层面：  
不是去预测像素，不是去重建视频，而是去预测**功能状态在干预条件下会如何迁移**。

这也是世界模型和状态解析引擎的本质区别：

- 状态解析回答：**现在在哪里**  
- 世界模型回答：**采取行动之后会往哪里走**

第三层还必须包含一件事：**记忆回写**。  
没有回写，就没有持续校准；没有持续校准，就没有真正意义上的世界模型。

## 六、三层的连接方式

这三层不是三个并列概念，而是一条递进路径。

可以用一句话概括：

> 功能协作关系层提供结构拓扑；  
> 状态解析引擎在该拓扑中定位当前状态；  
> 人体功能世界模型则在结构与状态之上模拟未来路径。

也可以更直接地说：

**定义结构 → 定位当前状态 → 模拟未来路径**

在执行层面，这种关系对应着一个更完整的闭环：

**观测输入 → 状态解析 → 安全过滤 → 候选行动 → 执行与复检 → 记忆回写 → 再次解析**

这和 MFLS 的闭环逻辑是一致的：  
记录层提供事实输入，安全层先过滤，知识层提供结构与规则，推理层输出结果，再回写到记录层形成纵向序列。:contentReference[oaicite:6]{index=6}

真正重要的是：  
这三层一旦连通，MUFO 就不再只是“知识治理系统”，而开始具备“动态决策系统”的雏形。

## 七、MUFO 在多尺度生物世界模型栈中的位置

如果把“世界模型”理解为一个从底层物理到高层决策的多尺度建模栈，那么 MUFO 的核心战略位置，并不在分子层，也不在纯临床结论层，而是在**人体功能动力学层**。

从工程角度看，这个分层可以写成：

- **L0 物理化学层**：原子、分子、力场、能量面  
- **L1 分子结构层**：蛋白折叠、配体构象、复合体结构  
- **L2 分子相互作用层**：结合、口袋、诱导契合、分子级相互作用  
- **L3 生物机制层**：通路、调控网络、细胞行为、组织响应  
- **L4 人体功能动力学层**：功能能力、代偿、稳定性、动作控制、功能边界、风险窗口  
- **L5 行为与任务层**：日常活动、运动表现、技能执行、作业能力  
- **L6 临床与决策层**：评估结论、干预路径、风险决策、管理策略  

这个分层真正承认的是：**尺度灾难是客观存在的**。  
不同层级的可观测性、噪声结构、控制变量和因果闭合方式完全不同，不可能被一个简单的统一模型端到端吞掉。LeCun 2022 所强调的多尺度抽象与分层规划，和这种认识是相容的。

MUFO 的真正靶心，是 **L4：人体功能动力学层**。

因为很多现有量表和系统，其实停在 **L5**，只回答：

- 能不能完成任务  
- 完成了多少  
- 表现如何  

而 MUFO 想往下压到 **L4**，去回答：

- 靠什么稳定域完成  
- 代偿是否启动  
- 风险边界在哪里  
- 下一步是该降阶、进阶，还是补测  

也正是在这一层，人体功能第一次不再只是行为结果的表面描述，也不再只是分子、机制与病理语言在上层的间接投影，而成为连接基础生命机制、真实任务表现与临床决策支持的关键中轴。人体功能动力学层既承接下层生物机制的约束，又向上支撑任务执行、训练调节与风险决策，因此它不仅是 MUFO 的战略落点，更是当前医学 AI 最有可能率先形成稳定坐标、建立可解释推理、实现责任闭环的现实落脚点。谁能够在这一层率先建立可计算、可审计、可演化的表达体系，谁就有机会真正把医学 AI 从“会生成答案”推进到“能理解人体状态、约束风险边界并支持长期决策”的新阶段。

## 八、为什么 MUFO 不等于普通知识图谱

MUFO 当然需要图结构，但 MUFO 不等于“图”本身。

普通知识图谱更擅长：

- 事实存储  
- 关系检索  
- 路径搜索  
- 实体连接  

这些都很有用，但它们通常不直接回答：

- 当前状态处于哪个功能区间  
- 哪条代偿路径已经被激活  
- 风险边界是否已被越过  
- 在当前护栏下下一步该做什么  

这也是为什么 MUFO 不能只停留在 graph schema。

W3C 的 OWL 2 为正式语义和推理提供了本体语言基础，JSON-LD 则为 JSON 系统与 linked data 之间提供了工程上的桥梁。  
但 MUFO 的真正目标，并不是“把知识图谱做大”，而是把**状态、风险、边界和演化**加入进去。

一个简化对比如下：

| 维度 | 普通知识图谱 | MUFO 目标 |
|---|---|---|
| 主产物 | 事实关系与路径 | 状态分布、风险窗口、演化路径 |
| 时间性 | 时间戳或事件堆叠 | 显式状态迁移与回写校准 |
| 约束 | 多为后置过滤 | 护栏前置、禁区优先 |
| 治理 | 常缺版本锁与推理证据 | 强调审计、溯源、版本与回放 |

所以，图只是 MUFO 的承载形式之一，而不是 MUFO 的终点。

## 九、为什么 MUFO 也不等于纯黑盒大模型

大模型可以写说明、给解释、做总结、生成候选建议，这些都有价值。  
但人体功能域不是一个可以完全依赖黑盒输出的领域。

真正构成信任的，不是“生成得像不像”，而是：

- 是否可回放  
- 是否可追溯  
- 是否可审计  
- 是否有明确安全护栏  
- 是否能承担责任闭环  

这也是为什么 MUFO 必须把“证据链”和“治理结构”放在核心位置。

W3C 的 PROV-O 提供了溯源表示框架，FHIR 的 AuditEvent 与 Provenance 分别覆盖审计事件和资源来源链条，为高要求场景中的追踪、可信度与责任界定提供了现实参考。

MUFO 与纯黑盒模型的差别，可以概括为：

| 维度 | 纯黑盒模型 | MUFO 推荐结构 |
|---|---|---|
| 输出稳定性 | 易受提示与上下文漂移影响 | 依赖版本化知识、规则与护栏 |
| 可追溯性 | 难以回放“为什么这样” | 保留证据链、规则版本与溯源对象 |
| 安全控制 | 多为后置修补 | 护栏前置、禁区优先 |
| 个体化 | 依赖提示注入 | 依赖纵向记录与回写校准 |

MUFO 不是拒绝大模型，而是明确一点：  
**大模型可以成为组件，但不能替代治理底座。**

## 十、MUFO 的工程价值、应用边界与长期意义

### 1. 工程价值

MUFO 的短期价值，不在“模拟一切”，而在于先把人体功能判断做成一个可重复、可审计、可治理的工程过程。

它可以直接带来：

- 记录统一  
- 术语边界稳定  
- 推理过程可追踪  
- 风险边界前置  
- 输出可复检  
- 多角色协作更一致  

这对于康复、训练、健康管理和功能教育都具有现实意义。

### 2. 应用边界

MUFO 必须始终保持边界清晰：

- 它不是诊断自动化  
- 它不是治疗自动化  
- 它不是无约束推荐生成系统  
- 它不是取消人工审核的黑盒替代物  

MUFO 是一个知识表示与决策支持框架。  
它支持判断，但不独占判断。

### 3. 中期意义

中期来看，MUFO 的真正价值在于形成一种**可校准的功能状态演化模型**。  
也就是说，不只是解释“现在为什么这样”，而是逐步积累“在什么条件下会向哪里变化”的经验与规则。

这时，世界模型层才真正开始发挥价值。

### 4. 长期意义

长期来看，MUFO 的意义不在于讲出一套更完整的话，而在于把人体功能从经验叙事推进到一个真正可治理的动态系统。

这意味着人体功能可以被：

- 统一表示  
- 结构化约束  
- 长期记录  
- 证据化推理  
- 安全边界控制  
- 纵向持续校准  

这也是为什么 MUFO 与 ICF、ICD-11、SNOMED CT 的关系应当是**映射与互操作**，而不是被外部体系反向定义。ICF 提供功能描述框架，ICD-11 提供疾病分类标准，SNOMED CT 提供临床术语体系；MUFO 的价值在于把“人体功能世界”本身建成一个可计算与可治理的中层表达。  
而 FHIR PlanDefinition 所代表的“计划作为可执行工件”的思路，也为 A10 干预路径和回退规则的表达提供了很好的工程参考。

### 5. 真正的成败标准

MUFO 最终能否成立，不取决于概念写得多漂亮，而取决于这几件事能不能做到：

1. 相同输入能否得到稳定且可解释的结果  
2. 结果能否把行动收敛到一两个真正关键的杠杆点  
3. 风险提示能否在复检中对应更差的耐受、恢复或进展  
4. 每一次评估、行动和回顾，能否都回写进系统，使后续判断更稳定  

如果这些做到了，MUFO 就不再只是一个概念架构，而会成为真正的人体功能决策基础设施。
